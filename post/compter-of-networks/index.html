计算机网络的二三事

Posted on 2020-01-10 Edited on 2020-06-29 In 日常技术 Views: Valine:

IP 地址

IP地址就是互联网的位置标记，就像我们现实中的地址一样，如果我们想给别人寄信，那我们首先要知道对方的地址，在互联网中就是 IP 地址。

IPv4

为了便于阅读，IPv4 采用了点分十进制，形如192.168.1.2，其二进制为11000000 10101000 00000001 00000010。
在 PHP 中的ip2long()与long2ip()函数可将点分十进制形式转换为整数形式，工作原理就是将二进制的值与权重相乘，如 1 * 2^31 + 1 * 2^30 + 1 * 2^23 + 1 * 2^21 + 1 * 2^19 + 1 * 2^8 + 1 * 2^1 = 3232235778。

NAT

随着互联网的普及，PC、移动设备与 IoT 的快速增长，IPv4 的紧缺日趋严重，在彻底的解决方案提出之前，网络地址转换（英语：Network Address Translation，缩写：NAT）被用做缓解 IPv4 的消耗。其原理是利用IPv4中的私网地址，将同一个私网中的所有设备共享同一个公网 IP 地址，比如，我们家庭中的路由器地址通常为 192.168.1.1，其中某台手机设备 IP 为 192.168.1.2，PC 设备的 IP 为 192.168.1.6，当手机或 PC 访问网络时，路由器会将手机或 PC 的数据包转换为

DNS

DNS 就是互联网的通讯录，它将难以记忆的 IP 地址与便于记忆的域名进行映射，使得我们可以轻松访问 Google、淘宝。同时，DNS 也承担了负载均衡的角色，比如，当伦敦与北京的用户访问Google时，他们虽然都输入了 www.google.com，但伦敦的用户得到 80.229.10.3，而北京的用户得到 172.217.26.142 的结果，两个 IP 地址分别指向了距离用户最近的服务器，不仅分摊了一个主机的流量，同时也让用户获得了更快速的响应体验。

DNS 默认使用 UDP 的 53 端口，因此监控者监控 53 端口的流量就可以嗅探用户所访问的网站，加密 DNS 在近几年的 HTTPS 普及中经常被提及，相关内容可以阅读 Cloudflare博客文章DNS加密说明 - Cloudflare

关于 DNS 的更多内容可以查看DNS 101

DDNS

由于 IPv4 的紧缺，我们大多数情况下都是通过 DHCP 方式获取随机分配的 IP，而域名需要绑定静态 IP，因此，动态DNS（英语：Dynamic DNS，简称DDNS）就是用来解决域名与动态 IP 绑定关系的，主机通过将自身当前的 IP 地址上报并修改与域名的绑定关系，从而使得非静态 IP 主机也能通过域名访问。

内网穿透

DHCP

发现家里路由器即使设置了 DHCP，但设备的 IP 仍几乎是固定的，比如树莓派的内网 IP 是192.168.1.134，一个月关机，重新打开时仍是192.168.1.134，经常观察，发现 IP 似乎并没有被重新分配，不知道这是否因为 DHCP 也有类似静态分配的机制。

DDoS

通常的DDoS攻击是攻击者控制大量僵尸电脑向目标主机发起洪流般的请求，直至目标无法响应；但新型的 DDoS 则是向目标主机注入恶意代码，当访问者访问网站时，也感染了这个病毒，然后所有受感染的访问者向目标主机发送大量请求，这被称为大炮 DDoS 攻击

SNI

我们将网络加密通信过程想象成一个邮政信封。信封里的东西是受保护的，邮差是看不到的。而信封外面是街道地址，邮差根据它来将信封送到正确的建筑位置。在互联网上，web服务器的IP地址就相当于街道地址。

然而，如果你住在一个多单元的大楼里，仅仅通过一个街道地址是不足以把信封送到正确的收件人手中的。为了补充街道地址，你要写上一个公寓号码或收件人的名字。这相当于SNI。如果web服务器承载多个域，SNI可以确保将请求路由到正确的站点，以便返回正确的SSL证书，从而能够加密和解密任何内容。
